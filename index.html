MoveRix Exception Dashboard

This dashboard shows how MoveRix AI processes logistics exception emails, classifies them, and routes them for human review when needed. It supports both admin and client views, with filtering, status updates, and mailbox-driven automation for exceptions such as detention, missing PODs, customs holds, delays, fleet issues, and cargo damage.

📊 Dashboard Overview

  • The dashboard lists all processed logistics exceptions in a table.
  • Each row includes key fields such as: id, customId, loadId, carrier, requested amount, status, email subject, resolution state, human override flag, email, and currency.
  • Example statuses shown in the system include: pending POD needs human approved customs hold issue damage reported delay

🔍 Filtering and Search

  • The sidebar provides filters for narrowing exceptions.
  • Available filters include: status, currency, date range, and load ID search.
  • Supported currencies shown in the demo are: USD, Kenyan Shillings (KES), EUR, and GBP.
  • The date range filter is used to view exceptions within a specific time window.
  • The load ID search helps locate a specific load when many records exist.

✅ Review Workflow

  • Exceptions that require attention appear in the pending review list.
  • Items marked needs human can be manually reviewed.
  • From there, an exception can be: approved or denied.
  • After a decision is made, the status updates in the table and the record moves into the appropriate filtered view.
  • The demo shows an item changing from needs human to approved after clicking the approve action.

⏱️ Detention Exception Example

  • A detention email with a 2-hour wait and a rate of $50/hour was automatically approved.
  • The AI assistant handled it because it matched the configured rules.
  • A second detention example with a 5-hour wait was routed to human review.
  • That item was then denied manually, and its status changed from needs human to denied.
  • The system also sends an email notification after the decision is made.

📄 Missing POD Workflow

  • A missing POD email was sent to the dedicated mailbox.
  • The system initially marked the case as pending POD.
  • After the proof of delivery was sent to the mailbox, the AI processed it asynchronously.
  • After processing, the status changes from pending POD to POD received.
  • The demo notes that this processing can take about 5 to 10 minutes.

⚙️ Processing Model

  • Users send exception emails to a dedicated mailbox.
  • The AI processes the mailbox in batches rather than instantly.
  • After waiting about 5 to 10 minutes, users log in to the dashboard to see updated statuses.
  • This is why the demo does not show immediate live processing for every sample.

👤 Admin vs Client Panel

  • The client panel has nearly the same functionality as the admin panel.
  • The main difference is that the admin-only controls are not shown in the client view.
  • Both views support: status filtering, currency filtering, date range filtering, load ID search, all exceptions view, full-screen table view, and download/export actions.
  • The client panel is scoped to the user’s own data.

🔒 Data Privacy and Access

  • Each client gets a dedicated mailbox and only sees their own exceptions.
  • Access requires the user’s email and password.
  • The presenter emphasizes that no one else can access the data unless credentials are shared.
  • This keeps each customer’s exception data isolated.

🚀 How to Use MoveRix

  • Send all exception emails to the dedicated mailbox.
  • Wait for the AI assistant to process them.
  • Log in to the dashboard to review: pending reviews, approved items, and human-reviewed items.
  • The system is designed to save teams significant time by automating exception handling.
